# SuperCo 技术使用说明 v1

> 版本: 2026-06-02
> 本文档介绍 SuperCo 平台的部署、配置和日常使用流程。

---

## 一、环境要求

| 组件 | 版本要求 | 用途 |
|------|---------|------|
| Go | 1.21+ | 编译运行服务端和 agent-runtime |
| PostgreSQL | 12+ | 数据持久化 |
| Node.js | 18+ | 前端构建和开发 |
| Claude Code | 最新 | AI 编程代理（可选） |

---

## 二、项目结构

```
superco/
├── server/            # Go 后端服务
│   ├── main.go        # 入口文件
│   ├── config/        # 配置加载
│   ├── database/      # 数据库连接与迁移
│   ├── handlers/      # HTTP + WebSocket 处理器
│   ├── middleware/     # JWT 认证中间件
│   ├── models/        # 数据模型
│   └── protocol/      # 消息总线协议（与 agent-runtime 共享）
├── agent-runtime/     # Agent 运行时
│   ├── runtime.go     # Runtime 主结构
│   └── backends/      # 后端实现（claude, echo 等）
├── webui/             # React 前端
│   └── src/
│       ├── App.tsx    # 主应用
│       ├── api/       # REST API 客户端
│       ├── components/# UI 组件
│       ├── hooks/     # React Hooks
│       ├── i18n/      # 国际化
│       └── types/     # TypeScript 类型定义
└── doc/               # 文档
```

## 三、快速开始

### 3.1 启动 PostgreSQL

```bash
# Windows 启动服务
net start postgresql-16

# 确认运行状态
psql -U postgres -c "SELECT version();"
```

### 3.2 配置环境变量

```bash
# server/.env 或系统环境变量
DATABASE_URL=postgres://postgres:password@localhost:5432/superco?sslmode=disable
JWT_SECRET=your-secret-key-change-in-production
```

### 3.3 启动服务端

```bash
cd server
go run main.go
# 服务监听 :8088
# 启动时自动创建数据库表（users, nodes, sessions, agents, agent_profiles）
```

### 3.4 启动 agent-runtime

```bash
cd agent-runtime
go run runtime.go --node-id node-1 --node-name "My Node"
# 连接到服务端 /ws/bus，注册能力（claude, echo）
```

### 3.5 启动前端开发服务器

```bash
cd webui
npm install
npm run dev
# Vite 开发服务器 :5173，代理 /api 和 /ws 到 :8088
```

### 3.6 访问 Web UI

打开浏览器访问 `http://localhost:5173`

1. **注册账号** — 首次使用需注册用户名和密码
2. **登录系统** — 输入已注册的账号信息
3. **查看节点** — 确认 agent-runtime 已连接（节点列表显示在线状态）

## 四、功能使用指南

### 4.1 节点管理

左侧导航栏点击 **节点** 进入节点列表页面：
- 查看所有已连接的 agent-runtime 节点
- 查看节点的在线/离线/忙碌状态
- 节点自动通过 Message Bus 注册和心跳

### 4.2 智能体管理

左侧导航栏点击 **智能体** 进入智能体页面：

**创建智能体：**
1. 点击 "+" 卡片
2. 填写智能体名称（必填）
3. 从下拉列表选择运行时（如 Claude Code、Echo）
4. 填写描述（可选）
5. 点击保存

**查看和编辑：**
1. 点击任意智能体卡片，弹出详情模态框
2. 查看名称、描述、运行时、版本、后端等信息
3. 点击"编辑"按钮修改名称和描述
4. 点击"删除"按钮删除智能体（需确认）

### 4.3 会话管理

左侧导航栏点击 **会话** 进入会话列表：
- 查看所有历史会话
- 点击会话进入聊天界面
- 右侧面板创建新的会话

**创建新会话：**
1. 在会话页面右侧选择目标节点
2. 点击创建
3. 自动跳转到聊天界面

### 4.4 聊天界面

主交互区域，支持：
- 发送文本消息
- 查看富消息渲染（文本、代码块、表格、进度条、工具调用等）
- `/clear` 命令清空消息
- 权限审批弹窗（受限模式下）

### 4.5 权限模式

聊天界面顶栏可切换权限模式：

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| **自动模式 (Auto)** | 自动批准所有工具调用 | 开发调试、高信任环境 |
| **受限模式 (Restricted)** | 逐条弹窗审批 | 生产环境、敏感操作 |

## 五、智能体配置 API

所有 API 需要 JWT 认证（`Authorization: Bearer <token>`）。

### 5.1 获取所有智能体

```
GET /api/agents/profiles
```

### 5.2 创建智能体

```
POST /api/agents/profiles
Content-Type: application/json

{
  "name": "代码助手",
  "description": "帮助编写和调试代码",
  "agent_id": "claude"
}
```

### 5.3 更新智能体

```
PUT /api/agents/profiles/:id
Content-Type: application/json

{
  "name": "新名称",
  "description": "新描述"
}
```

### 5.4 删除智能体

```
DELETE /api/agents/profiles/:id
```

### 5.5 获取可用运行时列表

```
GET /api/agents/runtimes
```

返回可用运行时能力列表（如 Claude Code、Echo）。

## 六、WebSocket 连接

### 6.1 Message Bus（聊天）

```
ws://localhost:8088/ws/bus?type=ui&user_id=<user>
```

使用信封协议进行实时通信。连接后发送 `hello` 信封：
```json
{
  "type": "hello",
  "from": "",
  "to": "system://bus",
  "payload": {
    "type": "ui",
    "user_id": "<user_id>"
  }
}
```

### 6.2 仪表盘（实时状态）

```
ws://localhost:8088/ws/dashboard?token=<jwt>
```

用于获取节点和会话的实时状态更新。

## 七、常见问题

### 7.1 服务端无法启动

检查 PostgreSQL 是否运行，数据库 URL 配置是否正确。

### 7.2 节点显示离线

确认 agent-runtime 已启动并正确连接到服务端地址。

### 7.3 WebSocket 连接失败

检查 Vite 代理配置（`vite.config.ts`），确保 `/ws` 路径正确代理到后端。

### 7.4 智能体创建失败

确保已至少有一个 agent-runtime 节点在线并注册了能力，且选择了正确的运行时类型。
