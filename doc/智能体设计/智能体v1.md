# 智能体卡片系统设计方案 v1

## 1. 背景

当前 Web UI 的"智能体"页面只是一个扁平的只读列表（`AgentList.tsx`），聚合显示各节点的 Agent 能力。用户希望将其改造为卡片式交互界面，让智能体成为独立的一级实体，支持创建、查看详情、编辑等操作。

---

## 2. 智能体定位

当前系统中存在两个概念：

| 概念 | 说明 |
|------|------|
| **Runtime 能力** | agent-runtime 连接时注册的能力（如 `claude`、`echo`），自动检测、不可手动创建 |
| **用户智能体配置** | 用户创建的智能体卡片，是对 Runtime 能力的「包装」——用户可以命名、写描述 |

**核心设计**：新增 `agent_profiles` 表（用户级，独立于节点），每个 profile 关联一个 runtime 能力 ID。用户通过"+"卡片创建 profile，系统通过 profile 找到支持该能力的 runtime 节点来执行会话。

---

## 3. 数据模型

### 新增表 `agent_profiles`

```sql
CREATE TABLE IF NOT EXISTS agent_profiles (
    id          VARCHAR(36) PRIMARY KEY,           -- 自动生成 UUID
    user_id     VARCHAR(36) NOT NULL REFERENCES users(id),
    name        VARCHAR(64) NOT NULL,               -- 用户命名，如"代码助手"
    avatar      VARCHAR(32) NOT NULL DEFAULT '🤖',  -- 自动生成，后期可自定义
    description TEXT NOT NULL DEFAULT '',            -- 用户描述，后期可用于规则/提示词扩展
    agent_id    VARCHAR(64) NOT NULL,               -- 对应 runtime 能力 ID（如 claude、echo）
    version     VARCHAR(32) NOT NULL DEFAULT '',
    model       VARCHAR(64) NOT NULL DEFAULT '',
    backend     VARCHAR(16) NOT NULL DEFAULT 'cli',
    enabled     BOOLEAN NOT NULL DEFAULT true,
    created_at  TIMESTAMP NOT NULL DEFAULT NOW(),
    updated_at  TIMESTAMP NOT NULL DEFAULT NOW()
);
```

### 与现有 `agents` 表的关系

- `agents` 表保留不变（节点级自动检测的 Agent 记录）
- `agent_profiles` 是用户级配置，通过 `agent_id` 字段关联到 runtime 能力
- 创建会话时，根据 profile 的 `agent_id` 找到可用的 runtime

## 4. 后端 API

新增 5 个接口（需 JWT 认证）：

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET`    | `/api/agents/profiles`       | 获取当前用户的所有智能体配置 |
| `POST`   | `/api/agents/profiles`       | 创建新的智能体配置 |
| `GET`    | `/api/agents/profiles/:id`   | 获取单个配置详情 |
| `PUT`    | `/api/agents/profiles/:id`   | 更新配置 |
| `DELETE` | `/api/agents/profiles/:id`   | 删除配置 |

新增文件：
- `server/handlers/agent_profile.go`
- `server/models/agent_profile.go`

## 5. 前端组件结构

```
webui/src/components/
├── AgentCard.tsx           ← 智能体卡片（3D 阴影，可点击展开）
├── AgentList.tsx           ← 重写：卡片网格布局 + inline 展开详情
├── AgentCreateCard.tsx     ← "+" 添加卡片（点击弹出创建 Modal）
└── AgentForm.tsx           ← 创建/编辑表单（Modal 和 inline 共用）
```

### 交互流程

```
智能体页面（卡片网格，每行 3-4 列，响应式）
  │
  ├── 卡片① [Claude Code]
  │     └── 点击 → 卡片 inline 展开详情
  │           ├── 显示：描述、头像、版本、状态、所属节点
  │           └── 点击编辑按钮 → 展开区域切换为编辑表单
  │                 ├── 字段：名称、描述、启用/禁用
  │                 └── 保存 → 折叠回卡片
  │
  ├── 卡片② [+]
  │     └── 点击 → 弹出 Modal 创建表单
  │           ├── 名称（必填）
  │           ├── 描述（可选，后期扩展为规则/提示词）
  │           └── 下拉选择 Runtime Agent 实体（如 Claude Code、Echo）
  │                 └── 确定 → 刷新列表，新卡片出现
  │
  └── ...
```

### 卡片 UI 设计

```
┌─────────────────────────────┐
│         🤖                   │
│    Claude Code               │  ← 名称（粗体）
│    AI 编程助手                │  ← 描述（小字）
│    ● 在线  |  v0.1.0        │  ← 状态 + 版本
│    Runtime: runtime-default  │  ← 所属节点
└─────────────────────────────┘

┌─────────────────────────────┐
│                             │
│              +              │  ← 大号加号（居中）
│                             │
│     添加新的智能体           │  ← 提示文字
│                             │
└─────────────────────────────┘
```
