## 二、组件通信方式

| 组件 | 连接方式 | 地址 |
|------|---------|------|
| Web UI → Server | WebSocket `/ws/bus?type=ui&user_id=xxx` | `ui://<user>/<conn>` |
| Web UI → Server | WebSocket `/ws/dashboard?token=jwt` | — |
| Web UI → Server | HTTP REST `/api/*` | — |
| agent-runtime → Server | WebSocket `/ws/bus?type=runtime` | `runtime://<node>` |

## 三、信封协议 (Envelope)

所有通信使用统一信封格式：

```json
{
  "id":        "msg_01HXYZ",
  "from":      "ui://user1/c1",
  "to":        "session://sess_abc",
  "type":      "message",
  "session_id":"sess_abc",
  "payload": {
    "content": [
      { "type": "text", "content": "Hello" }
    ]
  },
  "timestamp": 1717200000
}
```

### 地址系统

| 地址格式 | 说明 |
|----------|------|
| `ui://<user_id>/<conn_id>` | Web UI 客户端 |
| `runtime://<node_id>` | agent-runtime 实例 |
| `session://<session_id>` | 会话（广播给所有成员） |
| `system://bus` | 总线本身 |

### 消息类型

| 分类 | 类型 |
|------|------|
| 系统 | `hello`, `bye`, `ping`, `pong`, `ack`, `error` |
| 会话 | `session.create`, `session.created`, `session.join`, `session.joined`, `session.leave`, `session.end` |
| 应用 | `message`, `command`, `event`, `tool.use`, `tool.result` |
| 权限 | `permission.request`, `permission.response` |

## 四、会话生命周期

```mermaid
sequenceDiagram
    participant UI as Web UI
    participant B as Message Bus
    participant R as agent-runtime
    participant C as Claude CLI

    Note over UI,B: ① 创建会话
    UI->>B: session.create {agent:claude}
    B->>B: FindRuntimesForAgent(claude)
    B->>R: session.create {session_id, agent:claude}
    R->>C: 启动子进程 claude --stream-json
    C-->>R: Ready (stdout: system.init)
    R->>B: session.created {session_id}
    B->>UI: session.created {session_id}

    Note over UI,C: ② 消息交互
    UI->>B: message {to:session://xxx, text:"Hello"}
    B->>R: message {text:"Hello"}
    R->>C: stdin: {"type":"user","message":...}
    C-->>R: stdout: assistant event
    R-->>B: event {content_blocks}
    B-->>UI: event {content_blocks}

    Note over C: ③ 权限请求
    C-->>R: stdout: permission {tool_use_id, tool, input}
    R->>B: permission.request
    B->>UI: permission.request {弹窗}
    UI-->>B: permission.response {approved}
    B-->>R: permission.response
    R->>C: stdin: {"type":"approval","approved":true}

    Note over UI,C: ④ 结束会话
    UI->>B: session.leave
    B->>R: session.leave
    R->>C: 终止子进程
    R->>B: session.end
    B->>UI: session.end
```

## 五、权限审批流程

```mermaid
graph TD
    CLI["Claude CLI 需要工具权限<br/>(Bash/Edit/Read...)"] -->|"stdout: permission"| RT["agent-runtime"]
    RT -->|"permission.request<br/>信封协议"| BUS["Message Bus"]
    BUS -->|"转发到会话"| UI["Web UI"]

    UI --> Q{权限模式}

    Q -->|"Auto"| AUTO["自动批准<br/>→ permission.response {approved:true}"]
    Q -->|"Restricted"| DIALOG["PermissionDialog 弹窗<br/>显示工具名、参数、风险"]

    DIALOG -->|"Allow"| ALLOW["approved: true"]
    DIALOG -->|"Deny"| DENY["approved: false"]

    AUTO --> BUS2["Message Bus"]
    ALLOW --> BUS2
    DENY --> BUS2

    BUS2 --> RT2["agent-runtime"]
    RT2 -->|"stdin: approval"| CLI2["Claude CLI 继续执行"]

    style AUTO fill:#e8f5e9,color:#2e7d32
    style DIALOG fill:#fff3e0,color:#e65100
```
