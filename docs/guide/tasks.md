# 任务管理

任务是 CoAether 的工作单元。一个模糊的需求从创建到完成，经过智能体的层层处理。

## 创建任务

1. 在工作区中点击「新建任务」
2. 输入任务标题和描述
3. 可选：指定优先级、截止日期、指派人

### 任务属性

| 属性 | 说明 |
|------|------|
| 标题 | 任务的简要描述 |
| 描述 | 详细需求和上下文 |
| 优先级 | `low` / `medium` / `high` |
| 状态 | `todo` → `in_progress` → `review` → `done` |
| 指派人 | 指定负责的用户或智能体 |
| 截止日期 | 任务预期完成时间 |

## 任务分解

当任务被标记为需要分解（`auto_assign = true`），系统会：

1. 将任务交给「任务委派专家」智能体
2. 智能体分析任务内容，生成分解计划 (`decomposition_plan`)
3. 计划包含多个子任务，各有独立标题、描述、指派人
4. 子任务支持**依赖关系**（`depends_on`）和**并行分组**（`parallel_group`）
5. 审核通过后，子任务自动创建并进入执行队列

### 分解计划状态

```
pending → processing → approved / rejected
```

## 任务执行

子任务创建后进入智能体队列 (`task_agent_queue`)：

1. **队列等待**：匹配合适的智能体（能力匹配、负载检查）
2. **智能体认领**：智能体从队列取出任务开始处理
3. **执行**：智能体调用工具链完成任务
4. **审核门禁**：根据 `completion_behavior` 决定是否需要人工审核
5. **完成 / 返工**：审核通过则完成，否则进入返工循环

## 任务协作

### 评论

每个任务支持评论讨论，智能体也可以添加评论（`is_agent_comment = true`），便于追踪决策过程。

### 依赖关系

任务通过 `depends_on` 建立依赖链。前置任务完成后，后续任务自动从阻塞状态释放。

### 并行执行

同一 `parallel_group` 的任务可以同时执行，充分利用多智能体并发能力。

## 审核机制

审核师智能体负责对任务产出进行质量检查：

- 审核通过 → 任务完成
- 审核驳回 → 任务返回执行智能体进行修改（最多 `max_review_loops` 次）
- 超时 → 自动进入人工处理流程
