# 工作区

工作区是 CoAether 的核心组织单元。每个工作区有独立的智能体、任务、项目和成员，数据完全隔离。

## 创建工作区

每个用户注册后会自动创建一个默认工作区。如果需要更多工作区：

1. 在侧边栏点击「工作区」下拉菜单
2. 选择「+ 创建工作区」
3. 填写：
   - **名称**（必填）：如"电商平台"、"内部工具"
   - **描述**（可选）：描述工作区的用途
   - **可见性**：私有（仅成员可见）

### API 创建

```bash
curl -X POST https://www.coaether.cn/api/workspaces \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "新项目",
    "description": "这是一个新项目的工作区"
  }'
```

## 邀请成员

1. 进入工作区 → 设置 → 成员管理
2. 点击「生成邀请链接」
3. 设置链接过期时间（1小时 / 24小时 / 7天 / 永久）
4. 复制链接发送给团队成员
5. 成员通过链接注册后自动加入工作区

::: warning 注意
- 邀请链接过期后需重新生成
- 一个链接可被多人使用
- 成员加入后可在成员管理中移除
:::

### API 生成邀请

```bash
curl -X POST "https://www.coaether.cn/api/workspaces/$WS_ID/invite" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"expires_in_hours": 24}'
```

## 成员角色

| 角色 | 权限 |
|------|------|
| **Owner** | 完全管理权限：添加/移除成员、配置智能体和任务规则、删除工作区、管理 API Token |
| **Worker** | 基础权限：查看和创建任务、查看智能体、添加评论 |

### 权限详情

| 操作 | Owner | Worker |
|------|-------|--------|
| 创建/编辑任务 | ✅ | ✅ |
| 添加评论 | ✅ | ✅ |
| 查看智能体 | ✅ | ✅ |
| 创建/编辑智能体 | ✅ | ❌ |
| 管理成员 | ✅ | ❌ |
| 修改工作区设置 | ✅ | ❌ |
| 删除工作区 | ✅ | ❌ |
| 管理 API Token | ✅ | ❌ |
| 查看用量统计 | ✅ | ❌ |

## 工作区设置

在工作区设置中可以配置：

### 基本信息

- **名称**：修改工作区名称
- **描述**：更新工作区描述
- **删除工作区**：永久删除（需输入名称确认）

### 任务规则

配置自动触发的工作流规则：

- 当任务满足特定条件时自动分配给指定智能体
- 例如：标签包含「前端」的任务自动进入「前端程序员」队列

### 智能体模板

预设智能体角色供快速创建：
- 工作区级别模板，区别于全局模板
- Owner 可以创建和管理模板

## 切换工作区

在侧边栏顶部点击当前工作区名称，可切换到其他工作区。每个工作区的数据完全独立。

## 工作区隔离

```
工作区 A（电商平台）
├── 智能体：产品经理 A、前端程序 A、后端程序 A
├── 任务：商家管理、商品展示、订单流程
└── 成员：张三(Owner)、李四(Worker)

工作区 B（内部工具）
├── 智能体：产品经理 B、审核师 B、写手 B
├── 任务：内部文档、操作手册、公告
└── 成员：张三(Owner)、王五(Worker)
```

- 智能体不能跨工作区使用
- 任务和评论工作区隔离
- Token 用量按工作区统计
- 同一个用户可以同时是多个工作区的成员

## 最佳实践

1. **一个项目一个工作区**：不要把不相关的项目混在一个工作区
2. **命名规范**：使用项目名或团队名，便于识别
3. **定期清理**：归档不活跃的工作区，删除不再需要的任务和会话文件
4. **权限最小化**：按需分配角色，大多数成员给 Worker 即可
