# 智能体

智能体是 CoAether 中的 AI 工作者，每个智能体有独立的角色定义、能力声明和执行策略。

## 内置智能体

CoAether 提供了一套预定义的智能体角色：

| 智能体 | 角色 | 能力 |
|--------|------|------|
| 产品经理 | 将需求转化为 PRD | 需求分析、PRD 编写 |
| 产品需求挖掘经理 | 多轮对话引导需求澄清 | 需求挖掘、对话引导 |
| 任务委派专家 | 将 PRD 拆解为子任务 | 任务分解、依赖编排 |
| 写手 | 文案撰写与润色 | 文章撰写、内容创作 |
| 前端程序员 | 前端代码实现 | HTML/CSS/JS 编码 |
| 搜索师 | 信息检索与资料收集 | 网页搜索、知识查询 |
| 审核师 | 质量审核与评估 | 任务验收、质量把关 |
| 通用问题挖掘 | 通用入口，引导问题澄清 | 多轮对话、任务清单生成 |

## 自定义智能体

你可以创建自定义智能体：

1. 进入「智能体」页面
2. 点击「创建智能体」
3. 配置以下属性：

### 基础属性

- **名称和头像**：智能体的显示标识
- **描述**：智能体的角色说明
- **模型**：使用的大语言模型（Claude / GPT / 本地模型）

### 高级属性

- **系统提示词**：定义智能体的行为准则和专业知识
- **指令模板**：执行任务时的具体操作指引
- **能力声明** (`capabilities`)：智能体可使用的工具列表
- **最大并发** (`max_concurrency`)：智能体同时处理任务的上限
- **最大深度** (`max_depth`)：子任务拆解的最大层级
- **审核循环** (`max_review_loops`)：审核不通过时最大重试次数
- **完成行为** (`completion_behavior`)：任务完成后的默认操作（自动完成 / 需要审核）

## 智能体协议

CoAether 定义了一套标准协议来规范智能体间的协作。详见 [智能体能力声明与交互协议标准](https://github.com/madage/coaether)。

## 智能体负载

每个智能体的 `current_load` 表示当前正在处理的任务数。当负载达到 `max_concurrency` 时，新任务会进入队列等待。

你可以在管理后台查看所有智能体的实时负载状态。
